# Hermit v2.5 — Cross-Model Judgment Panel — Design Spec

**Status:** Approved (brainstorming, 2026-06-27)
**Parent:** [Hermit design spec](2026-06-26-hermit-design.md) → v2 verification-hardening layer (the moat's layer II: "decorrelate and weaponize the judges").

## Goal

Hermit's one LLM-*judged* signal is the intent match (v2.2: a single model decides "does this suite test the right goal?"). A single judge can be confidently wrong. v2.5 replaces it with a **panel** of different-tier models: their **consensus** is more robust than any one judge, and their **disagreement becomes a new uncertainty signal** that pulls the calibrated confidence down.

## Honest framing (stated up front)

Hermit is Anthropic-SDK-based, so the panel is different *tiers* (Opus / Sonnet / Haiku) — **partial** decorrelation, since they share a provider and training lineage. True cross-provider panels (different model families from different labs) are a future enhancement and the stronger form. But disagreement *within* the panel is still a real "the judges can't agree, so don't trust this verdict" signal — and the panel mean is a more stable estimate than a single draw.

## Method

For the intent judgment:
1. **Blind back-translate once** (a model reads the suite, infers the goal — unchanged from v2.2).
2. **Panel judge:** each model in the panel independently runs `judge_intent_match(original_goal, inferred_goal)` → a score + divergences.
3. **Aggregate:** consensus = mean of the scores; `agreement = max(0.0, 1.0 − (max − min))` (0 spread → 1.0; full spread → 0.0); a lone judge → agreement `1.0`. Divergences are unioned.

## Components

`hermit/panel.py`:

| Unit | Job |
|---|---|
| `PanelResult(mean: float, scores: dict[str, float], agreement: float)` | consensus + per-model scores + agreement |
| `aggregate_panel(scores: dict[str, float]) -> PanelResult` | pure, deterministic; `agreement=1.0` for ≤1 score, else `max(0, 1 − (max−min))`; empty → `PanelResult(0.0, {}, 1.0)` |
| `PanelIntentResult(score, agreement, inferred_goal, divergences, usage)` | `usage` = `list[tuple[model, in_tok, out_tok]]` (the back-translate call + each judge), for per-model budget charging |
| `panel_intent_check(goal, frozen_tests_dir, client, models) -> PanelIntentResult` | reads `test_*.py`, `back_translate` (via `models[0]`), `judge_intent_match` per model, `aggregate_panel`, unions divergences, collects per-model usage |

Reuses `back_translate` + `judge_intent_match` from `hermit.backtranslation` (no changes to those). Injectable client (the Anthropic client is model-agnostic — different `model` strings per call). Fully fake-tested.

## Integration (consensus signal + agreement floor — not a 4th confidence key)

The panel improves the **intent signal** and adds an **agreement floor**, rather than introducing a new weighted signal. This keeps the confidence formula at three keys (`holdout`/`mutation`/`intent`) — so every existing test stays green — and reuses the hard-floor mechanism already used for hold-out.

- **Intent signal = panel consensus.** When `config.intent_check_enabled and intent_client is not None and config.panel_enabled and len(config.panel_models) > 1`, the loop's intent hook calls `panel_intent_check(goal, frozen, intent_client, config.panel_models)` and feeds the **consensus mean** to the existing `intent` confidence signal — a strictly more robust estimate than v2.2's single judge. (When the panel is off or has one model, it falls back to the v2.2 single `run_intent_check`.)
- **Agreement floor.** The panel's `agreement` becomes a hard floor exactly like the hold-out floor: when gating is on and `agreement < config.panel_agreement_floor`, the run is forced to `low_confidence` regardless of the weighted average — "the judges can't agree, so don't trust this green." Escalation still applies.
- **Per-model budget charging:** each `(model, in, out)` in `pr.usage` is charged via `budget.charge_tokens`.
- `SolveResult.intent_score` holds the consensus; the run log records the agreement + per-model scores for transparency.
- `RunConfig` gains `panel_models: list[str] = ["claude-opus-4-8", "claude-sonnet-4-6", "claude-haiku-4-5"]`, `panel_enabled: bool = True`, `panel_agreement_floor: float = 0.5`. **`confidence_weights` is unchanged** (no 4th key).
- **Backward compatibility (clean):** existing loop tests pass a fake `intent_client` whose judge returns a fixed score regardless of model → all panel members return the same score → consensus = that score, agreement = 1.0 (≥ floor, no breach) → the 3-signal confidence and every existing assertion are **identical**. A 1-model panel also reduces to v2.2 exactly.

## Honest note

The agreement floor (a hard veto on judge disagreement) mirrors the hold-out floor: both are weakest-link guards that the weighted average can't smooth over. Consensus-as-intent and disagreement-as-veto together make the LLM-judged signal both more robust and more honest about its own uncertainty.

## Testing strategy

- `aggregate_panel`: pure — agreement for identical scores (1.0), spread (1 − spread), single score (1.0), empty (`(0.0, {}, 1.0)`), mean correctness.
- `panel_intent_check`: a fake client dispatching on `(output_format, model)` — `_InferredGoal` for back-translate, `IntentMatch` per judge model (different scores per model to exercise spread + per-model usage). Assert consensus, agreement, unioned divergences, and the per-model usage list.
- Loop: a fake `intent_client` whose judge returns the same score for all models → agreement 1.0 (≥ floor, no breach), consensus feeds `intent` → existing high/low-confidence tests stay green unchanged; plus a NEW test with a fake whose judge returns *different* scores per model → agreement below the floor → forces `low_confidence` even when the consensus + other signals are high.

## Out of scope (later)

- True cross-provider panels (OpenAI/Google/etc.) — the stronger decorrelation; needs multi-provider clients.
- Panel for *other* judgments (e.g. a panel mutation-equivalence judge) — the same `aggregate_panel` primitive would apply.
- The standalone `hermit intent-check` / `hermit verify` CLIs stay single-judge for now (the panel is the in-loop rigor where confidence is decided); they can adopt it later.
